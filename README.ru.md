# ScoreFlow 🎼

[English](README.md) · **Русский**

**Мобильный нотный редактор на Flutter и VexFlow.**

ScoreFlow позволяет создавать, редактировать, воспроизводить и печатать
партитуры прямо со смартфона — с упором на **фортепиано** (большой стан) и
**ударные**. Движок рендеринга VexFlow работает внутри WebView, звук — через
Web Audio, и всё это **полностью офлайн**: без аккаунта, без бэкенда, партитуры
хранятся локальными файлами.

В отличие от приложений «под гитарные табы», ScoreFlow заточен под клавишные и
перкуссию и выдаёт аккуратный постраничный **A4 PDF** без обрезанных тактов.

> Статус: **в активной разработке** — см. [ROADMAP.md](ROADMAP.md).

## Возможности

### Нотация
- Фортепианная нотация (скрипичный + басовый ключ, большой стан)
- Барабанная нотация (перкуссионный ключ, артикуляции через головки нот)
- Аккорды / многозвучные ноты
- Ноты и паузы с точкой
- 32-е и 64-е
- Лиги длительности (tie) и фразировки (slur)
- Tuplets — универсальное соотношение `actual:normal` (триоли, квинтоли, секстоли, септоли, произвольные)
- Знаки альтерации (♯ ♭ ♮ 𝄪 𝄫) — модель per-notehead; высота в playback = тональность + знак + правила такта
- Тональности и размеры тактов (включая нестандартные)
- Профессиональная бимовка (по долям, составные/нерегулярные размеры)
- Автодобивка такта (каноническая расстановка пауз)

### Воспроизведение
- Движок Web Audio (упреждающий планировщик)
- Сэмплированное фортепиано (Salamander Grand) с синтез-fallback
- Сэмплированная ударная установка с синтез-fallback
- Метроном
- Управление темпом (BPM)
- Follow Playback (автоскролл к текущей системе)
- Playhead, синхронизированный с нотами, + подсветка звучащих
- Демпфер-педаль (sustain)

### Редактирование
- Умная вставка (заполняет паузы / вставляет после курсора)
- Умное удаление (нота → пауза на месте)
- Undo / Redo (история снимков)
- Аккорд-режим
- Выделение диапазона (для лиг и tuplets)
- Выбор ноты тапом по партитуре

### Экспорт
- Постраничная вёрстка A4 (системная/страничная с выравниванием)
- Экспорт в PDF через системную печать

### Хранение
- Офлайн-first локальное хранилище (один JSON-файл на партитуру)
- Библиотека партитур (создание, открытие, переименование, удаление)

## Архитектура

```
Flutter (UI, состояние, файлы, транспорт)
   │  base64 JSON через evaluateJavascript / callHandler
   ▼
WebView (flutter_inappwebview)
   │  ассеты раздаёт локальный HTTP-сервер (офлайн)
   ▼
VexFlow (рендеринг нот, SVG)
   +
Web Audio API (сэмплированное воспроизведение + планировщик)
```

- **Локальный сервер** — `InAppLocalhostServer` отдаёт ассеты движка по
  `http://localhost`, поэтому относительные пути (`js/vexflow.js`, сэмплы)
  резолвятся.
- **Офлайн-ассеты** — движок VexFlow и звуковые сэмплы упакованы в приложение;
  сеть в рантайме не нужна.
- **JSON-транспорт** — партитура сериализуется в base64 JSON и передаётся в
  движок; тапы и события воспроизведения возвращаются через JS-мост.

## Запуск

### Требования
- Flutter SDK (Dart `>=3.12.2`)
- Android SDK 34+ (или тулчейн iOS)

### Запуск
```bash
git clone <repo-url>
cd scoreflow
flutter pub get
# Валидация зависимостей AGP отключена намеренно (см. tech debt в ROADMAP)
flutter run --android-skip-build-dependency-validation
```

### Тесты
```bash
flutter analyze
flutter test
```

### Звуковые сэмплы (опционально)
Сэмплы фортепиано и ударных уже лежат в репозитории, поэтому звук работает «из
коробки». Чтобы перегенерировать или заменить их, запустите одноразовые скрипты
(сеть нужна **только** при подготовке ассетов):

```bash
node tools/fetch_salamander.mjs   # фортепиано -> assets/www/piano/
node tools/fetch_drums.mjs        # ударные    -> assets/www/drums/
```

Если сэмплов нет, движок автоматически переходит на синтез-fallback.
